# 乐教库（LeJiaoKu）项目蓝图 · Agent 必读

## 一、项目定位

乐教库是一个面向全国教师与家长的长期教育资源平台。
核心目标：**让任何用户用最少步骤，最快找到可用教学资源**。

平台定位偏 TO B（教师 / 学校），兼顾 TO C（家长）。
未来支持订阅制（Free / ¥10 / ¥20）。

---

## 二、核心原则（不可违反）

1. 教育语义优先于技术实现
2. 简单优先，不为炫技增加复杂度
3. 所有结构支持长期维护
4. 单人可维护是硬性约束
5. 不允许隐式业务规则

---

## 三、资源语义模型（极其重要）

### 资源的三层语义，严禁混用

#### 1️⃣ 教学用途（Category）
- 教材
- 教案
- 课件
- 习题
- 视频
- 其他

👉 用于 **首页分类 / 主筛选**

---

#### 2️⃣ 文件格式（FileFormat）
- PPT
- PDF
- DOC
- 图片
- 视频

👉 仅用于技术信息展示 / 下载判断

---

#### 3️⃣ 教学属性（Tag / Metadata）
- 学科（语文 / 数学 / 英语…）
- 年级（1–6 / 7–9）
- 教材版本（人教版 / 北师大版）
- 目录节点（章 / 节）

👉 用于精确筛选与推荐

---

## 四、用户角色

- admin：审核 / 管理
- contributor：上传资源（需审核）
- subscriber：付费订阅
- user：普通用户

---

## 五、技术约束

- 后端：Node.js + Express + MySQL
- 前端：Vue（不强制版本）
- 单体架构
- 不引入微服务
- 不引入过度中间件

---

## 六、Agent 行为约束

- 不擅自新增业务概念
- 不修改已有语义定义
- 不假设“后面会重构”
- 所有实现必须能落库
